# KOVCHEG Blog 3.3 — Growth Suite

Growth Suite добавляет инструменты распространения публикаций и поддержания поисковых адресов.

## Адреса

- `/sitemap.xml` — карта публичных материалов;
- `/robots.txt` — правила индексации;
- `/feed.xml` — RSS публикаций;
- `/studio/growth` — управление SEO, редиректами, подписчиками и публикацией.

## Публикация по расписанию

Материал со статусом `scheduled` публикуется, когда наступает его `published_at`.

Команда:

```bash
php bin/publish-scheduled.php
```

Для FASTPANEL или системного cron рекомендуется запуск раз в пять минут от пользователя сайта:

```cron
*/5 * * * * cd /var/www/kovchegcms_r_usr/data/www/kovchegcms.ru && /usr/bin/php bin/publish-scheduled.php >> storage/logs/scheduled-publish.log 2>&1
```

Команда обрабатывает до 200 материалов за один запуск и записывает каждую автоматическую публикацию в `content_publication_log`.

## Редиректы

Исходный адрес задаётся внутренним путём, например `/old-page`.

Назначение может быть:

- внутренним путём `/new-page`;
- полным адресом `https://example.com/page`.

Поддерживаются коды 301, 302, 307 и 308. Для постоянного переноса используйте 301 или 308.

## Подписки

Подписка сохраняет email и источник формы. В 3.3 реестр предназначен для подготовки аудитории и последующего подключения почтового модуля. Массовая отправка писем в ядро не встроена.

## SEO

Перед включением индексации проверьте:

1. название и описание сайта;
2. публичность нужных материалов;
3. корректность меню и canonical URL;
4. отсутствие тестовых страниц;
5. доступность sitemap и RSS.
