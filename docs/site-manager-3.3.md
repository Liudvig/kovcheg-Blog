# KOVCHEG Blog 3.3 — Site Manager

## Назначение

Site Manager переносит управление главной страницей, SEO и взаимодействием с читателями в KOVCHEG Studio. Владелец сайта меняет структуру и настройки без редактирования PHP-файлов темы.

## Главная страница

Раздел: `/studio/homepage`.

Доступные секции:

- первый экран;
- последние записи;
- избранный материал;
- портфолио;
- текстовый блок;
- показатели;
- призыв к действию.

Для каждой секции доступны:

- внутреннее название;
- служебный ключ;
- позиция;
- включение и отключение;
- перемещение вверх и вниз;
- параметры, зависящие от типа секции.

Удаление секции не удаляет публикации, страницы или работы портфолио.

## SEO-центр

Раздел: `/studio/seo`.

Основные функции:

- общий суффикс заголовка;
- описание сайта по умолчанию;
- изображение Open Graph по умолчанию;
- Google site verification;
- Яндекс verification;
- управление индексацией;
- включение RSS и sitemap;
- дополнительные директивы robots.txt;
- редиректы 301 и 302.

Системные адреса:

- `/robots.txt`;
- `/sitemap.xml`;
- `/feed.xml`.

### SEO материала

В редакторе публикации доступны:

- SEO-заголовок;
- SEO-описание;
- canonical URL;
- Open Graph-тип;
- отдельное изображение для социальных сетей;
- запрет индексации конкретного материала.

При пустых индивидуальных значениях используются настройки сайта и обложка материала.

## Обсуждения

Читатель может:

- подписаться на обсуждение;
- ответить на конкретный комментарий;
- поставить реакцию комментарию;
- изменить или удалить свой комментарий в разрешённый период;
- открыть личные уведомления по адресу `/notifications`.

Время редактирования задаётся настройкой `comments_edit_minutes`. По умолчанию — 30 минут. Модератор, администратор и владелец не ограничены этим временем.

Уведомления создаются для:

- пользователей, подписанных на обсуждение;
- автора комментария, на который поступил ответ.

## Редиректы

Исходный адрес должен начинаться с `/`. Целевой адрес может быть внутренним путём или полным URL `http/https`.

Поддерживаются:

- `301` — постоянный перенос;
- `302` — временный перенос.

Studio показывает количество срабатываний каждого редиректа.

## Миграция

Файл:

```text
migrations/20260722_blog_site_manager.sql
```

Миграция создаёт:

- `site_home_sections`;
- `content_subscriptions`;
- `content_comment_reactions`;
- `blog_notifications`;
- `seo_redirects`.

Также она добавляет SEO-поля материалам и поле времени редактирования комментария. Все условные изменения схемы используют `information_schema`, поэтому повторный запуск безопасен на MariaDB и MySQL.

## Обновление

Перед обновлением сохраните базу данных, `storage/uploads` и `config/config.php`.

```bash
git fetch origin main
git reset --hard origin/main
php bin/migrate.php
```

После обновления проверьте:

```text
/
/studio/homepage
/studio/seo
/sitemap.xml
/feed.xml
/robots.txt
/notifications
```
