# Безопасность

## Обязательные меры

- используйте HTTPS;
- используйте актуальную поддерживаемую версию PHP;
- выдайте пользователю базы права только на базу KOVCHEG CMS;
- не публикуйте `config/config.php`, дампы, журналы, кеш, загрузки и резервные копии;
- регулярно проверяйте восстановление резервной копии;
- ограничьте доступ к административной панели и удаляйте неиспользуемые API-токены и Webhook.

`config/config.php` содержит пароль базы и ключ приложения. Ключ приложения используется для шифрования защищённых настроек и не должен меняться без контролируемой процедуры миграции.

## Apache

В поставку включены `.htaccess`, закрывающие служебные каталоги и чувствительные типы файлов. На сервере должны быть разрешены правила `AllowOverride`.

## Nginx

Если корень сайта направлен на каталог проекта, запретите прямой доступ к:

- `/config/`;
- `/database/`;
- `/storage/`;
- `/app/`;
- `/routes/`;
- `/views/`;
- `/docs/`;
- `/modules/`;
- скрытым файлам и дампам.

Пользовательские файлы должны выдаваться только через маршруты приложения, проверяющие права доступа.

## После установки

Проверьте раздел «Админка → Диагностика и безопасность», настройте cron, создайте тестовую резервную копию и убедитесь, что `config/config.php` отсутствует в Git.
