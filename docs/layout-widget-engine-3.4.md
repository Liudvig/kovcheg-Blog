# KOVCHEG Blog 3.4 — Layout & Widget Engine

Layout & Widget Engine позволяет управлять структурой публичного сайта из KOVCHEG Studio без изменения PHP-файлов темы.

## Открытие редактора

Перейдите в:

```text
Studio → Виджеты и зоны
```

Прямой адрес:

```text
/studio/widgets
```

Раздел доступен владельцу и администратору.

## Основные понятия

- **Виджет** — настроенный блок: меню, поиск, текст, публикации, рубрики и т. п.
- **Зона** — место вывода: шапка, левая колонка, правая колонка, содержимое или подвал.
- **Схема** — полный набор размещений для определённого контекста сайта.
- **Не размещены** — виджеты, которые сохранены, но не показываются на сайте.
- **Ревизия** — снимок расположения перед очередным сохранением.

## Добавление виджета

1. Найдите нужный тип в блоке «Добавить виджет».
2. Нажмите `+`.
3. Новый экземпляр появится в разделе «Не размещены».
4. Перетащите его в нужную зону.
5. Откройте «Настройки» внутри карточки.
6. Сохраните настройки экземпляра.
7. Нажмите «Сохранить расположение».

Создание экземпляра ещё не выводит его на сайт. Для вывода требуется разместить карточку в зоне и сохранить схему.

## Перемещение

Карточку можно:

- перетащить мышкой;
- поднять кнопкой `↑`;
- опустить кнопкой `↓`;
- убрать с сайта кнопкой «Убрать из зоны».

После перестановки обязательно нажмите «Сохранить расположение».

## Несколько меню

Одно и то же меню можно вывести в нескольких местах:

1. создайте виджет «Меню»;
2. выберите нужное меню в настройках;
3. скопируйте экземпляр;
4. разместите одну копию в `header.main`, другую — в `layout.left` или `footer.columns`.

Для разных мест можно использовать разные созданные меню.

## Перенос и отключение поиска

Поиск является обычным виджетом `core.search`.

Его можно:

- поставить в шапку;
- перенести в левую или правую колонку;
- поставить над содержимым;
- временно отключить;
- убрать в «Не размещены»;
- полностью удалить.

Тема не содержит обязательного жёстко встроенного поиска.

## Включение и отключение

Кнопка «Отключить» скрывает экземпляр, сохраняя:

- настройки;
- расположение;
- связь с меню;
- данные модуля.

После повторного включения блок вернётся на прежнее место.

## Копирование

Кнопка «Копировать» создаёт новый независимый экземпляр с такими же настройками. Это удобно для:

- одного меню в шапке и подвале;
- нескольких текстовых блоков;
- разных списков публикаций;
- повторного использования социальных ссылок.

## Ревизии

Перед каждым сохранением расположения система записывает предыдущую схему.

Для отката:

1. откройте список ревизий справа;
2. выберите нужную дату;
3. нажмите «Восстановить»;
4. подтвердите действие.

Восстановление также создаёт новую ревизию, поэтому последнее состояние не теряется.

## Стандартные зоны

```text
header.top
header.main
header.bottom
page.before
layout.left
content.before
content.after
layout.right
page.after
footer.top
footer.columns
footer.bottom
```

`content.main` зарезервирована для содержимого открытой страницы и не принимает обычные виджеты через Studio.

## Базовые виджеты

### Логотип и название

Показывает системный логотип, название и при необходимости подпись сайта.

### Меню

Выводит любое меню, созданное в разделе `Studio → Меню`.

### Поиск

Отправляет запрос на читаемый адрес `/search?q=...`.

### Профиль и вход

Для гостя показывает вход и регистрацию. Для авторизованного пользователя — профиль и, при наличии прав, Studio.

### Текстовый блок

Выводит безопасный текст. HTML и JavaScript не выполняются.

### Изображение

Выводит изображение, ALT-текст и необязательную ссылку. Разрешены внутренние пути и HTTP/HTTPS URL.

### Последние публикации

Автоматически получает новые опубликованные записи.

### Рубрики

Показывает рубрики и количество привязанных материалов.

### Подписка

Использует существующий Growth Suite и сохраняет email в реестре подписчиков.

### Социальные ссылки

Формат каждой строки:

```text
Название | https://example.com/profile
```

## Модули и собственные виджеты

Модуль может зарегистрировать тип:

```php
\Kovcheg\Blog\Layout::registerWidget('example.weather', [
    'label' => 'Погода',
    'description' => 'Погодный виджет модуля.',
    'module' => 'example-weather',
    'defaults' => ['city' => 'Воронеж'],
    'fields' => [
        'city' => [
            'label' => 'Город',
            'type' => 'text',
            'maxlength' => 120,
        ],
    ],
    'render' => static function (array $settings): string {
        return '<div>'.e($settings['city']).'</div>';
    },
]);
```

Модуль также может объявить дополнительную зону:

```php
\Kovcheg\Blog\Layout::registerZone('weather.panel', [
    'label' => 'Панель погоды',
    'width' => 'sidebar',
]);
```

Тема должна явно вывести дополнительную зону, если она является уникальной для этой темы.

## Отказоустойчивость

- ошибка renderer одного блока записывается в журнал;
- остальные блоки и страница продолжают работать;
- неизвестный тип не выводится, но запись остаётся в базе;
- отключение модуля не удаляет настройки и размещение;
- до применения миграции тема использует старую безопасную шапку и подвал.

## Обновление с 3.3.3

После получения кода выполните:

```bash
php bin/migrate.php
```

Затем откройте `/studio/widgets`, проверьте начальную схему и сохраните её.
