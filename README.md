# KOVCHEG Blog 3.4

KOVCHEG Blog — модульный движок для авторского блога, новостного сайта, портфолио и профессионального сайта. Он построен на ядре KOVCHEG CMS и использует готовые механизмы безопасности, пользователей, ролей, профилей, модулей и системных сервисов.

Версия **3.4.0** добавляет Layout & Widget Engine: владелец управляет шапкой, колонками, центральным содержимым и подвалом из KOVCHEG Studio. Меню, поиск, профиль, списки публикаций и другие блоки можно убрать, переместить, настроить или скопировать без правки PHP-шаблона.

Следующая цель — **KOVCHEG Blog 4.0**: отдельная трёхколоночная Portal Theme, Module SDK 3, безопасные Embed-провайдеры, расширенные URL/SEO и подключаемая социальность.

## Принципы развития

- **Module first:** новые функции преимущественно создаются как модули/плагины;
- ядро содержит контракты, безопасность, реестры, права, маршрутизацию и lifecycle;
- поиск, меню, профиль, подписка и другие блоки можно убрать, переместить или продублировать;
- темы отвечают за зоны, сетку и оформление, а не за жёстко встроенную бизнес-логику;
- публичные адреса должны быть читаемыми и сохранять SEO при изменении slug;
- SEO работает независимо от активной темы;
- комментарии, реакции, подписки, уведомления и другие социальные функции подключаются независимо;
- внешнее видео и содержимое встраиваются через безопасный Embed Provider API.

## Layout & Widget Engine 3.4

Системные зоны:

- `header.top`, `header.main`, `header.bottom`;
- `page.before`, `page.after`;
- `layout.left`, `layout.right`;
- `content.before`, `content.after`;
- `footer.top`, `footer.columns`, `footer.bottom`.

В разделе **Studio → Виджеты и зоны** доступны:

- drag-and-drop между зонами;
- кнопки перемещения вверх и вниз;
- удаление из зоны без удаления настроек;
- включение и отключение экземпляра;
- копирование экземпляра;
- отдельные настройки каждого блока;
- ревизии расположения и восстановление;
- публикация всей схемы одним действием.

Базовые виджеты:

- логотип и название;
- любое созданное меню;
- поиск;
- профиль и вход;
- безопасный текст;
- изображение;
- последние публикации;
- рубрики;
- подписка;
- социальные ссылки.

Модуль может добавить собственный тип через `Kovcheg\Blog\Layout::registerWidget()` и собственную область через `Layout::registerZone()`.

## Требования

- PHP 8.3 рекомендуется, минимально PHP 8.1;
- MySQL 8 или MariaDB 10.6+;
- PDO MySQL, mbstring, fileinfo, JSON и OpenSSL;
- GD или Imagick для обработки изображений;
- ZipArchive для модулей и архивов;
- cURL для Webhook и внешних интеграций;
- HTTPS для рабочего сайта.

## Чистая установка

1. Загрузите содержимое репозитория в каталог сайта.
2. Создайте пустую базу MySQL/MariaDB и отдельного пользователя.
3. Откройте `install.php`.
4. Завершите мастер установки.
5. Примените миграции:

```bash
php bin/migrate.php
```

6. Переименуйте или удалите `install.php`.
7. Откройте `/studio/widgets`, настройте расположение и сохраните схему.

## Публикация по расписанию

```bash
php bin/publish-scheduled.php
```

Рекомендуемый cron:

```cron
*/5 * * * * /usr/bin/php /path/to/site/bin/publish-scheduled.php >> /path/to/site/storage/logs/publish-scheduled.log 2>&1
```

## Обновление установленного сайта

Перед обновлением сохраните базу данных, `storage/uploads` и `config/config.php`.

```bash
git pull --ff-only origin main
php bin/migrate.php
```

Полная инструкция для FASTPANEL находится в [`docs/deployment-fastpanel.md`](docs/deployment-fastpanel.md).

## Безопасность

- конфигурация, пользовательские файлы, журналы, кеш и резервные копии исключены из Git;
- служебные каталоги закрыты от прямого веб-доступа;
- публичные медиа отдаются через контролируемый PHP-шлюз;
- формы защищены CSRF;
- пароли хешируются Argon2id, если он доступен;
- загрузки проверяются по размеру и MIME-типу;
- SQL выполняется через подготовленные выражения;
- неизвестный или отключённый модульный виджет не вызывает fatal error публичной страницы;
- renderer каждого виджета изолирован обработчиком ошибок;
- произвольный HTML и внешние iframe не должны обходить sanitizer, allowlist и CSP;
- CI проверяет PHP, JavaScript, JSON, секреты, MariaDB, MySQL, миграции, runtime, Widget Engine и публичные маршруты.
