# KOVCHEG Blog 3.2

KOVCHEG Blog — быстрый движок для авторского блога, сайта-портфолио и профессионального сайта. Он построен на ядре KOVCHEG CMS и сохраняет готовые механизмы безопасности, пользователей, ролей, профилей, модулей и системных сервисов.

Версия 3.2 добавляет Visual Builder: владелец собирает страницу из готовых блоков, применяет профессиональный пресет, управляет медиатекой, пользователями и модулями из единой панели KOVCHEG Studio.

## Основные возможности

- публикации блога, обычные страницы и работы портфолио;
- Visual Builder с 20 типами блоков;
- готовые секции лендинга, кейса, музыкального релиза и страницы услуги;
- сохранение собственных шаблонов секций;
- перетаскивание, дублирование, автосохранение и предпросмотр;
- текст, заголовки, изображения, галереи, аудио, видео, цитаты, списки, колонки, кнопки, код, первый экран, показатели, этапы, отзывы, карточки и контакты;
- черновики, приватные материалы и публикация по расписанию;
- история изменений и восстановление ревизий;
- рубрики и теги;
- SEO-заголовок и описание для каждого материала;
- обложки и защищённая медиатека с папками;
- комментарии с предварительной модерацией;
- реакции читателей и жалобы на комментарии;
- регистрация, личный профиль и кабинет пользователя;
- роли: владелец, администратор, редактор, модератор, пользователь и гость;
- управление ролями и блокировками в Studio;
- настраиваемое меню сайта;
- темы KOVCHEG Editorial и KOVCHEG Portfolio;
- профессиональные пресеты музыканта, строителя, художника и разработчика;
- безопасные модули формата 2;
- статистика просмотров;
- панель управления **KOVCHEG Studio** по адресу `/studio`.

## Требования

- PHP 8.3 рекомендуется, минимально PHP 8.1;
- MySQL 8 или MariaDB 10.6+;
- PDO MySQL, mbstring, fileinfo, JSON и OpenSSL;
- GD или Imagick для обработки изображений;
- ZipArchive для модулей и архивов;
- cURL для Webhook и внешних интеграций;
- HTTPS для рабочего сайта.

## Чистая установка

1. Загрузите содержимое репозитория в каталог сайта.
2. Создайте пустую базу MySQL/MariaDB и отдельного пользователя.
3. Откройте `install.php`.
4. Завершите мастер установки.
5. Выполните миграции:

```bash
php bin/migrate.php
```

6. Переименуйте или удалите `install.php`.
7. Откройте `/studio` и настройте сайт.

## Обновление установленного сайта

Перед каждым обновлением сохраните базу данных, каталог `storage/uploads` и файл `config/config.php`. Затем получите новую версию из Git и примените миграции:

```bash
git pull --ff-only origin main
php bin/migrate.php
```

Полная инструкция для FASTPANEL находится в `docs/deployment-fastpanel.md`.

## Безопасность

- `config/config.php`, пользовательские файлы, журналы, кеш и резервные копии исключены из Git;
- служебные каталоги закрыты от прямого веб-доступа;
- публичные медиа отдаются через контролируемый PHP-шлюз;
- формы защищены CSRF;
- пароли хешируются Argon2id, если он доступен;
- загрузки проверяются по размеру и MIME-типу;
- SQL выполняется через подготовленные выражения;
- модульный ZIP проверяется на traversal, symlink и запрещённые исполняемые файлы;
- CI проверяет PHP, JavaScript, JSON, секреты, чистую схему MariaDB и все миграции.
