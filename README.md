# KOVCHEG CMS 3.0 — Ядро

Чистое ядро социальной CMS-платформы KOVCHEG: профили, стена и лента, личные сообщения, комментарии, реакции, уведомления, роли, API, Webhook, резервные копии, PWA и три шаблона интерфейса.

## Что входит

- шаблон `default`;
- светлый шаблон `vk` в стиле классической социальной сети;
- шаблон `x` для мультиблогового формата;
- профили, аватары, баннеры, публикации и медиатека;
- личные сообщения, вложения, голосовые сообщения, реакции и стикеры;
- роли владельца, администратора, редактора, пользователя и гостя;
- административная панель, диагностика, резервное копирование и сборка установочного архива;
- REST API, Webhook с HMAC-подписью, Web Push и планировщик;
- мастер чистой установки.

## Требования

PHP 8.1+, MySQL/MariaDB, PDO MySQL, mbstring, fileinfo, JSON, OpenSSL и ZipArchive. Для обработки изображений требуется GD или Imagick. Для безопасной отправки Webhook требуется cURL. Рабочий сайт должен использовать HTTPS.

## Установка

1. Загрузите содержимое репозитория в отдельный каталог сайта.
2. Создайте пустую базу MySQL/MariaDB и пользователя базы.
3. Откройте `install.php` в браузере.
4. Заполните параметры базы и создайте владельца.
5. После установки войдите в административную панель и настройте регистрацию, роли, резервные копии и cron.

Секретный файл `config/config.php`, пользовательские загрузки, журналы, кеш, дампы и резервные копии не должны попадать в Git или публичные архивы.

## Безопасность

- каталоги `config`, `database`, `storage`, `app`, `routes`, `views`, `docs` и `modules` закрыты `.htaccess`;
- запросы к базе выполняются через подготовленные выражения;
- формы защищены CSRF-токенами;
- пароли хешируются Argon2id, если он доступен;
- ключи и секреты шифруются ключом приложения;
- загрузки проверяются по размеру, MIME-типу и допустимым форматам;
- Webhook проверяются на публичный адрес и отправляются без редиректов;
- установочные архивы исключают runtime-данные и проходят повторную проверку.

Для Nginx необходимо отдельно запретить прямой веб-доступ к служебным каталогам. Подробности находятся в `docs/security.md`.
